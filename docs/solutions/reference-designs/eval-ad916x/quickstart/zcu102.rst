ZCU102 Quickstart
===============================================================================

This guide provides quick instructions on how to set up the
:adi:`EVAL-AD916x` on:

- :xilinx:`ZCU102 <ZCU102>` Rev 1.0 or later

.. image:: ../../images/zcu102.jpg
   :width: 900

.. esd-warning::

Using Linux as software
-------------------------------------------------------------------------------

Necessary files
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

.. note::

  The SD card includes several folders in the root directory of the BOOT
  partition. In order to configure the SD card to work with a specific FPGA
  board and ADI hardware, several files must be copied onto the root directory.
  Using the host PC, drag and drop the required files onto the BOOT partition,
  and use the EJECT function when removing the SD card from the reader.

The following files are needed for the system to boot:

- HDL boot image: ``BOOT.BIN``
- Linux Kernel image: ``uImage``
- Linux device tree: ``devicetree.dtb``

They can either be taken from the SD card -- already generated by us, or you
can build them manually:

- Instructions on how to create the SD card image can be found in the
  :dokuwiki:`ZynqMP image creation workflow <resources/tools-software/linux-software/zynq_images>`.
- For the BOOT partition, you will need the following files specific to AD916x + ZCU102:

  - ``Image`` file for ZynqMP (from zynqmp-common folder)
  - ``BOOT.BIN`` specific to your AD916x + ZCU102
  - ``system.dtb`` specific to your AD916x + ZCU102

.. important::

  Some projects provide multiple devicetree files in the SD card's boot
  folders. Make sure you select the devicetree that matches your specific use
  case.

Required Software
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

- SD Card imaged with :dokuwiki:`ZynqMP image <resources/tools-software/linux-software/zynq_images>`
- A UART terminal (Putty/Tera Term/Minicom, etc.) with baud rate 115200 (8N1)
- :git-iio-oscilloscope:`IIO Oscilloscope <releases>` (for remote operation)

Required Hardware
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

- :xilinx:`ZCU102 <ZCU102>` Rev 1.0 or later board with power supply (12V)
- :adi:`EVAL-AD916X-FMCZ <EVAL-AD916X>` FMC evaluation board
- SD card with at least 8GB of memory (16GB recommended)
- 1 x Micro-USB cable (for UART console)
- Ethernet cable (for network access)
- USB keyboard and mouse
- Display Port compatible monitor

Testing
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Creating the setup
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

EVAL-AD916X-FMCZ/ZCU102
"""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""

.. image:: ../images/ad9164fmc+zcu102.png
   :width: 900

In the following example, we will prepare the evaluation setup by connecting the
:adi:`EVAL-AD916X-FMCZ` board to the ZCU102.

Follow the steps in this order, to avoid damaging the components:

#. Get the :xilinx:`ZCU102 <ZCU102>` and all required accessories
#. Insert the SD card into the SD Card Interface Connector (J100)
#. Connect the :adi:`EVAL-AD916X-FMCZ` FMC board to the FPGA carrier HPC0 FMC0 socket
#. Plug your Display Port monitor into the Display Port Video Connector (P11)
#. Plug your USB mouse/keyboard into the USB 2.0 ULPI Controller, w/Micro-B Connector (J83)
#. Plug the Power Supply into 12V Power input connector (J52) (DO NOT turn the device on)
#. Set the jumpers of SW6 (Boot Mode) to: 1: on, 2: off, 3: off, 4: off

    .. image:: ../../images/sw6_zcu102.jpg
        :width: 800

#. Turn on the power switch on the FPGA board
#. Wait ~30 seconds for the "DONE" LED (near the SD Card Interface Connector) to turn green
#. Observe kernel and serial console output messages on your terminal

Useful commands for the serial terminal
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

The below commands are to be run in the serial terminal connected to the FPGA.

**Login Information**

user: analog
password: analog

To find out the IP of the FPGA board, run the following command and take the
IP specified at "eth0 inet" (when using remote IIO Oscilloscope):

.. shell::

   $ifconfig

To see the IIO devices detected, run:

.. shell::

    $iio_info | grep iio:device
    iio:device0: ad916x (buffer capable)
    iio:device1: xadc

To power off the system, run the following command, and wait for the final
message to be printed, then power off the FPGA board from the switch as well.

.. shell::

   $poweroff

To reboot the system, run:

.. shell::

   $reboot

.. important::

   Even though this is Linux, this is a persistent file system. Care should
   be taken not to corrupt the file system -- please shut down things, don't
   just turn off the power switch. Depending on your monitor, the standard
   power off could be hiding. You can do this from the terminal as well with
   :code:`sudo shutdown -h now` or the above-mentioned command for powering
   off.

Scopy
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

.. important::
   Make sure to download/update to the latest version of
   :external+scopy:doc:`Scopy <index>`.

:external+scopy:doc:`Scopy <index>` can be used to drive the DAC outputs of the
evaluation board. Refer to the :adi:`EVAL-AD916X` documentation for output signal
configuration and test procedures.

Refer to the IIO Oscilloscope or Scopy documentation for detailed configuration
instructions specific to the AD916x device and your desired output mode.

IIO Oscilloscope
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

.. important::
   Make sure to download/update to the latest version of
   :git-iio-oscilloscope:`IIO Oscilloscope <releases>`.

#. Once done with the installation or an update of the latest IIO Oscilloscope,
   open the application. You can use either local or remote connection:

   - **Local connection**: Use the display connected to the ZCU102 board directly
   - **Remote connection**: Connect via Ethernet to the ZCU102 board IP address

#. For remote operation, disconnect the USB mouse/keyboard from J83 and connect a
   Micro-USB UART cable to J83 to communicate with the board.
#. Press ``Refresh`` to display available IIO Devices and press ``Connect``.
#. After the board is connected and a channel is enabled, hit the play button.
   You should then see the data capture window displaying the AD916x output.

.. Setup
.. -------------------------------------------------------------------------------

.. Create/configure the SD card
.. ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

.. #. Prepare SD card using ADI ZynqMP image flow.
.. #. Ensure BOOT partition contains:

..    - `BOOT.BIN` for AD916x + ZCU102
..    - `Image` from zynqmp-common image set
..    - `system.dtb` matching AD916x/ZCU102 variant

.. #. Safely eject SD card and insert into ZCU102 J100.

.. HDL build
.. ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

.. Build the default configuration:

.. .. code-block:: bash

..    cd hdl/projects/ad916x_fmc/zcu102
..    make

.. Default project configuration targets AD9162 mode 8.

.. Build examples for alternate device/mode selections:

.. .. code-block:: bash

..    cd hdl/projects/ad916x_fmc/zcu102
..    make ADI_LANE_RATE=12.5 ADI_DAC_DEVICE=AD9163 ADI_DAC_MODE=02

.. .. code-block:: bash

..    cd hdl/projects/ad916x_fmc/zcu102
..    make ADI_DAC_DEVICE=AD9164 ADI_LANE_RATE=12.5 M=1 L=8 S=4 NP=16

.. Clocking and jumper notes
.. ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

.. Use J61 to select clock source on the evaluation board:

.. - J61 mounted: on-board clock generator
.. - J61 unmounted: external clock source

.. For high-lane-rate modes or specific device/mode combinations, verify clock
.. requirements in the AD916x HDL project documentation before bring-up.

.. Hardware setup (ZCU102)
.. ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

.. #. Connect EVAL-AD916X-FMCZ to HPC0.
.. #. Connect Micro-USB to J83 (UART).
.. #. Connect Ethernet cable to P12.
.. #. For local GUI flow, connect DisplayPort and USB keyboard/mouse.
.. #. Set ZCU102 SW6 to SD boot mode per board guide and ADI quickstart references.
.. #. Apply power at J52 and switch board on.
.. #. Wait for DONE LED indication and boot progress on UART.

.. Bring-up sequence
.. ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

.. #. Apply required clock source configuration (J61 and external clock if needed).
.. #. Program HDL design.
.. #. Boot Linux or no-OS stack.
.. #. Confirm target IP via UART (`ifconfig` / `ip addr`).
.. #. Connect using host IIO-Oscilloscope (remote) or local desktop app.
.. #. Verify JESD link status and DAC output tone generation for selected mode.

.. Validation checklist
.. ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

.. - JESD link reports synchronized/aligned state
.. - SPI access to AD9508, ADF4355, and AD916x device is functional
.. - DMA/offload data path streams expected waveform
.. - Measured DAC output matches selected frequency/mode settings

.. Troubleshooting checklist
.. ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

.. - No UART output: verify J83 port selection, 115200 8N1, and SW6 SD boot mode
.. - No network IP: check Ethernet link and DHCP availability
.. - JESD not in DATA: verify clock source/jumpers and selected JESD mode
.. - Wrong output tone: confirm selected DDS/DMA source and NCO configuration
